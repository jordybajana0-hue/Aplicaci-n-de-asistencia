# Propuesta de Mantenimientos
La aplicación de control de asistencia requiere diferentes tipos de mantenimiento para asegurar su correcto funcionamiento, mejorar sus características y adaptarse a nuevas necesidades. A continuación se presentan las propuestas de mantenimiento recomendadas:
# Mantenimiento Correctivo
Este mantenimiento se aplica cuando aparecen errores o fallas durante el uso de la aplicación.
- Ejemplos propuestos:
Corregir errores al registrar la entrada o salida de usuarios.
Solucionar fallas en el cálculo de horas trabajadas.
Arreglar mensajes de error incorrectos.
Reparar problemas en la visualización del historial.
# Mantenimiento Preventivo
Se realiza para evitar que ocurran fallas futuras y mantener estable el sistema.
- Ejemplos propuestos:
Revisar periódicamente la base de datos para evitar registros duplicados.
Optimizar consultas para que la aplicación responda más rápido.
Verificar la integridad de los archivos de configuración.
Limpieza automática de registros muy antiguos para mejorar rendimiento.
# Mantenimiento Perfectivo
Busca mejorar el sistema agregando nuevas funciones o mejorando las existentes.
- Ejemplos propuestos:
Agregar filtros más avanzados en el historial de asistencia.
Mejorar la interfaz para que sea más intuitiva.
Incluir exportación de reportes en PDF o Excel.
Añadir notificaciones cuando un usuario registre llegada tarde.
# 4. Mantenimiento Adaptativo
Se aplica cuando el sistema debe ajustarse a cambios externos.
- Ejemplos propuestos:
Adaptar la aplicación a nuevas políticas internas de asistencia.
Modificar la base de datos si cambia la estructura institucional.
Actualizar compatibilidad con nuevas versiones del sistema operativo.

Ajustar el sistema para nuevas normativas laborales o académicas.
